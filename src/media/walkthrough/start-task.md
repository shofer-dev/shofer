# Start Your First Task

Once Shofer is open and you've selected a mode and provider, you're ready to go.

## How It Works

1. Type a message in the chat input bar
2. Shofer reads your workspace files, searches code, runs commands, and edits files
3. Review changes in the **File Changes Panel** — accept or revert each one
4. The **Context Window Bar** shows how much conversation capacity remains

## Example First Tasks

- "Explain what this project does"
- "Add unit tests for the `UserService` class"
- "Refactor the authentication middleware to use async/await"
- "Find and fix the bug where users can't reset their password"

## While Shofer Is Working

- **Queue messages** — keep typing while Shofer processes; messages queue up automatically
- **Click Stop** — cancel the current operation immediately
- **Click Send Now** — jump your queued message to the front

Click **New Task** in the step description to create your first task.
